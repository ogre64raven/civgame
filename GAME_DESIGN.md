# 온라인 문명 — 게임 설계 문서

버전 1.0 (2026-07-13) · 30인 동시접속 실시간 턴제 문명 게임 · **구현 완료된 현행 규칙 기준**

## 1. 개요

| 항목 | 내용 |
|---|---|
| 동시접속 | 최대 30명, 단일 월드 |
| 국가 | UN 회원국 193개국 중 중복 없이 배정 (수도가 5타일 미만 소형 섬인 28개국 제외 → 165개국). 첫 플레이어는 랜덤, 이후는 **기존 플레이어 수도들과의 최소 거리가 가장 먼 나라**를 자동 선택 |
| 국가코드 | ISO 3166-1 alpha-3 (KOR, USA, JPN …) |
| 맵 | 실제 세계지도 기반 헥스 그리드 135×68 (육지 2,889타일 ≈ 31.5%), 동서 랩어라운드 |
| 턴 구조 | 회의 턴 30초 → 실행 턴 10초 반복 (관리자가 변경 가능), 기본 120턴 제한 |
| 시작 조건 | 문명당 유닛 3기 + 수도 1타일 영토 |
| 자원 | 돌 · 곡식 · 목재 · 철 |
| 서버 | Node.js + ws WebSocket, 인메모리 상태 (서버 권위 방식) |
| 클라이언트 | WebGL(Three.js r128, 로컬 번들) 3D 렌더링, 의존성 최소화 |
| 배포 | GitHub → Docker 이미지 → 리눅스 컨테이너 |

## 2. 아키텍처

```
[브라우저 WebGL 클라이언트]
        │  WebSocket (JSON)
        ▼
[Node.js 게임 서버 (서버 권위)]
  ├─ Game   : 턴 타이머·명령 큐·실행 턴 리졸버·외교·승리 판정 (server/game.js)
  ├─ World  : 헥스맵·지형·인접·경로 보조 (server/world.js)
  ├─ Bots   : 봇 AI — 회의 턴마다 연구·생산·확장 (server/bots.js)
  └─ Net    : HTTP 정적 서빙 + 관리자 API + WS 프로토콜 (server/index.js)
```

- 클라이언트는 명령만 보내고 모든 판정은 서버가 수행 (치팅 방지). 상태는 인메모리, 서버 재시작 시 새 게임.

## 3. 맵과 지형

- odd-r offset·pointy-top 헥스, 동서로 이어짐(랩). `npm run buildmap`으로 재생성.
- 모든 육지 타일은 자원을 하나씩 가지며 맵에 아이콘(곡식 다발·바위·나무·철광석)으로 표시.

| 지형 | 자원 | 이동 비용 | 비고 |
|---|---|---|---|
| 바다 `~` | — | 3 | 이동 가능, 영토화 불가 |
| 초원 `g` | 곡식 | 1 | |
| 채석지 `p` | 돌 | 1 | |
| 숲 `f` | 목재 | 1 | |
| 구릉 `h` | 철 | 1.5 | 산지 1단계 (~10%) |
| 산 `m` | 철 | 2 | 산지 2단계 (~7%) |
| 고산 `M` | 철 | 3 | 산지 3단계 (~5%), 설산 렌더링 |

- **스폰**: 배정 국가의 실제 수도 좌표에서 가장 가까운 육지에 수도 + 유닛 3기. 수도가 있는 대륙이 5타일 미만이면 그 국가는 배정 풀에서 제외.

## 4. 턴 구조

```
[회의 턴 30초] 명령 입력·수정, 연구/생산 예약, 채팅, 외교        (봇도 이때 명령)
      ▼
[실행 턴 10초] 입력 잠금 → 서버가 동시 해석 → 결과 브로드캐스트 + 연출
```

실행 턴 해석 순서: ① 스턴 감소 → ② 이동(영토 점령·보물 획득) → ③ 전투 → ④ 접촉 갱신 → ⑤ 수도 공성 → ⑥ 채취(동맹 배분) → ⑦ 연구 → ⑧ 유닛 생산 → ⑨ 외교(파기·흡수) → ⑩ 승리 판정.

## 5. 유닛과 이동

- **이동력 턴당 3** (+ 이동 기술 3레벨마다 +1). 지형별 비용은 §3 표 참조.
- 목표 클릭 → 비용 기준 최단 경로(다익스트라) 자동 산출. **연속 클릭으로 웨이포인트** 연장, X/ESC로 취소.
- 경로는 여러 턴에 걸쳐 자동 진행. 이동 라인은 두꺼운 3D 경로로 표시.
- **요격**: 실행 턴 이동은 전 유닛이 한 칸씩 번갈아 전진하는 동시 시뮬레이션 — 이동 중 적 유닛과 같은 헥스에서 만나면(정면 교차·주둔지 통과) 양측 그 자리에서 멈추고 그 턴 전투가 벌어진다. 패자는 남은 경로가 취소된 채 후퇴.
- 유닛은 소멸하지 않음 — 전투 패배 시 근처로 후퇴할 뿐 (§7).

## 6. 영토와 자원

- 유닛이 지나간 육지는 그 문명의 **영토**가 됨. 동맹 영토는 밟아도 뺏지 않음.
- **영토가 매 실행 턴 타일당 해당 자원 1을 자동 채취** (채집 기술 +20%/Lv).
- **동맹 수입 배분**: 동맹 그룹의 수입을 합산해 **영토 수 비율**로 배분 (최대잉여법으로 정수 보존, 채집 기술은 각자 자기 영토분에 적용).
- 자기 영토 + 동맹 영토는 이어진 색면과 문명색 외곽 테두리로 표시.

## 7. 전투

같은 헥스에서 적대 유닛 조우 시 (동맹끼리는 전투 없음·공존 가능):

- 세력(동맹 연합) 단위로 전투력 비교: **유닛별 군사 레벨 합 + 수 우위 + 자기 영토면 인구 레벨** 보정.
- 승자: 헥스 유지 + 해당 헥스 점령. 패자: **전투 지점에서 2헥스 떨어진 빈 육지(자기 수도 방향 우선)로 후퇴 + 행동불능** (소멸 없음). 적 수도 공격에 실패해도 근처에서 재정비해 다시 노릴 수 있음 (퇴각지가 없으면 수도로).
- **패배 누적**: 유닛별로 질 때마다 행동불능이 길어짐 — 1패 2턴, 2패 3턴, 3패 4턴 … 다음 두 가지로 초기화:
  - **군사 기술 향상**(연구·보물) 시 그 문명의 전 유닛 누적 리셋.
  - **회복 명령**: 유닛당 곡식 10 소모 → 현재 행동불능 즉시 해제 + 누적 초기화 (유닛 창의 '회복' 버튼).
- 무승부: 양측 모두 2턴 행동불능(누적 없음). 행동불능 유닛만 있는 헥스에서는 전투 재발동 없음.
- 클라이언트에서 대결·승리·패배·무승부 연출 표시.

## 8. 수도 공성과 예속

- 수도 HP = **5 + 4계통 기술 레벨 총합** (최대 27). 공격이 없는 턴 회복 = **기본 1 + 최대 레벨(5) 도달 계통마다 +1** (최대 5).
- 수도 위 적 유닛이 매 실행 턴 **유닛당 (1+군사Lv) 피해**.
- **HP 0 → 함락**: 유닛·영토가 점령국에 편입되고 패자는 **예속** 상태로 위임 유닛을 조작하며 계속 플레이. **위임 수는 점령국이 예속국별 1~3기 선택**.

## 9. 유닛 생산

- 유닛 상한 = **3 + 인구 기술 레벨** (최대 8기).
- 회의 턴에 예약(**곡식 10 + 돌 10**) → 실행 턴에 수도에서 생성. 봇도 자동 생산.

## 10. 기술 (4계통 × 5레벨 · 군사만 7레벨, 비용 20×목표레벨)

| 계통 | 자원 | 효과 |
|---|---|---|
| 군사 | 철 | 전투력 +1/Lv, 공성 피해 +1/Lv, 유닛 외형 진화(석기→현대군→**근미래(Lv6)→미래(Lv7)**) — 유일하게 7레벨까지 |
| 인구 | 곡식 | 유닛 상한 +1/Lv + 자기 영토 전투 +1/Lv |
| 채집 | 목재 | 채취량 +20%/Lv |
| 이동 | 돌 | 3레벨마다 이동력 +1 |

회의 턴 예약 → 실행 턴에 자원 차감·완료. 수도 건물이 기술 총합에 따라 6단계 진화(스톤헨지→마천루).

## 10.4 요새 (장성)

- 회의 턴에 **자기 유닛이 서 있는 자기 영토**(수도·기존 요새 제외)에 예약 → 실행 턴에 완공. 비용 **돌 10 + 목재 10**.
- HP = 건설 시점 **수도 최대 HP의 절반(올림)** (기본 3, 기술 만렙 시 14). 공격이 없는 턴 +1 회복.
- **적 유닛은 요새 타일을 통과·점령할 수 없음** — 경로는 자동 우회되고, 막히면 앞에서 대기하며 자동 공성. 인접한 적 유닛이 턴당 (1+군사Lv) 피해, HP 0 → 파괴.
- 중립 유닛(야생동물·부족)도 요새 타일에 진입 불가 → 영토 보호선 역할.
- 인접한 같은 문명의 요새·수도끼리 **성벽으로 이어져 긴 장성**으로 렌더링 (망루+깃발+총안).
- 수도 함락 시 요새도 점령국에 편입.

## 10.5 중립 유닛 (야생동물 · 원시 부족)

- 게임 시작 시 **문명별 고립도에 따라 배치** (늑대·곰·호랑이·사자·원시 부족, 수도에서 6헥스 이상·서로 4헥스 이상 이격): 반경 14헥스 내 상대 수도가 **0개면 그 수도 주변(12헥스 이내)에 9마리, 1개면 6마리, 2개 이상이면 3마리** — 상대가 적은 외딴 스타트일수록 PvE 위협이 커짐.
- 매 실행 턴 **인접 육지로 1칸 배회** (수도 헥스는 밟지 않음). **밟은 플레이어 영토는 해제**되어 소유가 풀림.
- **지나칠 수 없음**: 이동 경로에 중립 유닛이 있으면 그 헥스에서 정지하고 교전한다. 유닛이 올라선 헥스의 중립 유닛도 도망가지 않고 맞선다.
- 강함은 **항상 초기 문명 유닛과 동일(군사 0)이고 성장하지 않음**. 같은 헥스의 문명 유닛과 교전:
  - 유닛 측이 우위(군사 기술·2기 이상 수 우위·자기 영토 방어)면 **중립 유닛 처치** (행동불능 중립 유닛도 처치됨).
  - 동률(군사 0 유닛 1기)이면 양측 2턴 행동불능.
- 영토를 점령하지 않고, 수도 공성·보물 획득도 하지 않음. 안개 밖에서만 보임.

## 11. 보물상자

- 게임 시작 시 **참가 인원 ÷ 2개**를 배치. 후보 점수 = 모든 수도로부터의 거리(최소 8) + 험지 가중(고산+8·산+4·구릉+2) + 12타일 이하 섬 +8. 상자끼리도 8헥스 이상 이격.
- 유닛이 밟으면 획득: **자원 상자(전 자원 +20)** 또는 **기술 상자(원하는 계통 +1레벨 선택 팝업)**. 봇은 가장 낮은 기술 자동 선택.
- 금색 상자 + 빛 링으로 렌더링, 안개 속에서는 보이지 않음.

## 12. 외교

- **접촉**: 유닛이 상대 유닛·영토·수도의 2헥스 이내로 접근하면 영구 기록. **동맹은 접촉한 문명끼리만 가능**.
- 동맹 제안 → 수락 시 성립. **그룹 최대 3개국** — 2국 동맹에서 확장하려면 나머지 동맹국의 동의 팝업 승인 필요.
- 동맹 효과: 전투 불발·시야 공유·수입 배분(§6)·공동 승리.
- 파기는 선언 후 다음 실행 턴 발효. **위약금**: 파기 신청측이 발효 시점 자원의 **10%를 상대에게 지급** (3국 동맹이면 자기 동맹국들에게 **5%씩**, 종류별 내림).
- **흡수**: 동맹 쌍의 점수(기술 합 + 영토 수) 비율이 **8:2 초과 3턴 연속**이면 약자가 강자에게 흡수(예속과 동일).

## 13. 승리 조건

1. **제패**: 예속 아닌 문명이 1개만 남음 (동맹이면 공동 승리).
2. **점수**: 제한 턴(기본 120턴) 도달 시 점수(기술 합 + 영토 수) 최고 문명 승리, 동맹 공동 승리.

## 14. 시야와 안개

- 자기 유닛·영토·수도 기준 **3헥스** + 동맹 시야 공유. 그 밖은 구름 낀 안개 기둥으로 가림.
- 게임 종료·관전자는 전체 맵 공개.

## 15. 로비 · 관리자 · 봇

- **방 번호**: 게임(리셋)마다 4자리 방 번호가 생성되며, 입장 시 방 번호가 맞아야 접속 가능 (재접속 토큰은 생략). 관리자 페이지에 방 번호와 **접속 QR**(방 번호가 채워진 입장 URL)이 표시됨. 테스트/운영 고정은 `ROOM` 환경변수.
- 입장하면 국가 배정 후 **로비 대기**. 게임 진행 중 새 입장자는 즉시 참전, 정원 초과·종료 후에는 관전자.
- 재접속: 세션 토큰으로 기존 문명 복귀. 접속이 끊겨도 문명은 유지.
- **강퇴 유저 재접속**: 강퇴된 국가는 배정 풀로 복귀하며, 해당 유저는 방 번호만 다시 입력하면 **새 국가로 재입장** 가능 (만료 세션은 자동 정리·안내).
- **관리자 페이지 `/admin`** (비밀번호: `ADMIN_PASSWORD` 환경변수, 기본 admin1234): 시작·리셋·강퇴·회의/실행 시간·턴 제한 설정·봇 추가/제거·**맵 관전**(안개 없는 전체 맵 실시간 관찰).
- **봇**: 회의 턴마다 가장 낮은 기술 연구, 여유 자원 시 유닛 생산, 오래 묶인 유닛은 곡식으로 회복, 자원이 넉넉하면 국경에 요새 건설, 주변 비소유 타일로 확장(보물 우선, 군사 0이면 중립 유닛 회피, 적 장성 우회), 군사 우위일 때만 적 수도 공격. 동맹 제안은 수락하고 확장 동의 요청은 승인.
