# 온라인 문명 — 게임 설계 문서

버전 0.2 (2026-07-13) · 30인 동시접속 실시간 턴제 문명 게임

> **v0.2 영토전 개편 요약** (구현 반영 완료)
>
> - 유닛은 문명당 3기 고정 — 생산·소멸 없음. 전투 패자는 소멸 대신 **수도로 후퇴 + 2턴 행동불능**.
> - **이동력 기본 2** (이동 기술 3Lv마다 +1). 육지 1칸 = 이동력 1, **바다 1칸 = 이동력 2** (바다는 턴당 1칸). 경로는 비용 기준 최단(다익스트라). 바다는 영토화되지 않음.
> - 유닛이 지나간 육지는 그 문명의 **영토**가 됨 (동맹 영토는 존중). 전투 승자는 전투 헥스 점령.
> - 모든 육지 타일은 자원을 가지며(아이콘 표시), **영토가 매 실행 턴 자동 채취** (채집 기술 +20%/Lv).
> - **수도 헥스를 빼앗기면 점령(예속)** — 유닛·영토가 점령국에 편입되고 위임 유닛으로 계속 플레이. **위임 유닛 수는 점령국이 예속국별로 1~3기 중 선택** (참가국 목록의 위임 드롭다운).
> - **동맹 수입 배분**: 동맹 그룹의 채취 수입을 합산해 매 실행 턴 **영토 수 비율로 배분** (자원 종류가 섞여 분배됨, 채집 기술은 각자 자기 영토분에 적용).
> - **동맹은 최대 3개국**. 2국 동맹에서 확장하려면 **나머지 동맹국의 동의**가 필요하며, 해당 플레이어에게 동의/거부 팝업이 표시됨.
> - 기술 4계통: 군사(철)·**방어(고기, 자기 영토 전투 +1/Lv)**·채집(목재)·이동(곡식). 비용 20×레벨.
> - 문명 점수 = 기술 레벨 합 + 영토 수. 흡수(8:2, 3턴)·승리 조건은 이 점수 기준.
> - 로비 시스템: 입장 후 대기, **관리자(/admin)가 시작·리셋·강퇴·턴 시간 설정**.
> - 바다 이동이 없으므로 본토와 단절된 고립 섬나라 23개국은 배정 제외 (170개국 배정).
>
> 아래 본문은 v0.1 원안이며, 충돌 시 위 요약이 우선한다.

## 1. 개요

| 항목 | 내용 |
|---|---|
| 동시접속 | 최대 30명, 단일 월드 |
| 국가 | 실제 UN 회원국 193개국 중 랜덤·중복 없이 배정 |
| 맵 | 실제 세계지도 기반 헥사곤(HEX) 그리드, 바다/육지 구분 |
| 턴 구조 | 회의 턴 30초 → 실행 턴 10초 반복 |
| 시작 조건 | 문명당 인구유닛 3기 |
| 자원 | 고기 · 곡식 · 목재 · 철 |
| 서버 | Node.js + WebSocket, 인메모리 상태 (서버 권위 방식) |
| 클라이언트 | HTML5 Canvas 2D, 의존성 최소화 |
| 배포 | GitHub → Docker 이미지 → 리눅스 컨테이너 |

## 2. 아키텍처

```
[브라우저 Canvas 2D 클라이언트]
        │  WebSocket (JSON)
        ▼
[Node.js 게임 서버 (서버 권위)]
  ├─ GameLoop     : 회의/실행 턴 타이머, 페이즈 전환
  ├─ World        : 헥스맵, 타일 자원
  ├─ Civilization : 국가, 유닛, 자원, 기술, 외교 상태
  ├─ Resolver     : 실행 턴에 모든 명령 동시 해석 (이동→전투→채취→연구)
  └─ Net          : 접속/재접속, 상태 브로드캐스트
```

- 클라이언트는 명령만 보내고, 모든 판정은 서버가 수행 (치팅 방지).
- 상태는 인메모리. 서버 재시작 시 새 게임 시작.
- 30명 × 초당 수 회 메시지 수준이므로 단일 Node 프로세스로 충분.

### 디렉터리 구조(예정)

```
/server
  index.js        # HTTP + WebSocket 부트스트랩
  gameLoop.js     # 턴 타이머·페이즈
  world.js        # 맵 생성·타일
  civ.js          # 문명·유닛·자원·기술
  resolver.js     # 명령 동시 해석
  protocol.js     # 메시지 정의·검증
/client
  index.html
  main.js         # 입장·소켓·상태 동기화
  render.js       # Canvas 2D 헥스맵 렌더링
  input.js        # 유닛 선택·명령 UI
/data
  countries.json  # UN 193개국 (이름·코드·수도 좌표)
  worldmask.json  # 육지/바다 헥스 마스크
Dockerfile
.github/workflows/deploy.yml
```

## 3. 맵

- **좌표계**: axial 좌표 (q, r), pointy-top 헥스.
- **크기**: 기본 90 × 45 (약 4,050타일, 육지 약 30%). 30명 기준 육지 1인당 ~40타일.
- **생성**: 등장방형(equirectangular) 세계지도 육지 마스크를 헥스 그리드에 샘플링 → `worldmask.json`으로 빌드 타임에 미리 생성. 서버는 로드만 함.
- **타일 종류**: 바다(이동 불가) / 초원(곡식) / 숲(목재) / 평원(고기) / 산(철). 육지 타일에 위도·지형 기반으로 자원 배치.
- **스폰**: 배정받은 국가의 수도 좌표에서 가장 가까운 빈 육지 헥스에 유닛 3기 배치. 인접 스폰 간 최소 거리 보장.

## 4. 턴 구조

```
[회의 턴 30초] 명령 입력·수정, 채팅, 동맹 제안/수락
      ▼
[실행 턴 10초] 입력 잠금 → 서버가 명령 동시 해석 → 결과 브로드캐스트 + 연출
      ▼ (반복)
```

실행 턴 해석 순서: ① 이동 → ② 전투 판정 → ③ 채취 → ④ 연구/생산 → ⑤ 외교(흡수 판정 포함).

- 유닛 이동력: 턴당 1헥스 (이동 기술로 증가 가능).
- 같은 헥스로 서로 다른 문명 유닛이 진입하면 전투 발생.

## 5. 자원과 채취

- 유닛을 자원 타일로 보내면 그 턴부터 자동 채취. 턴당 기본 1 (채집 기술로 증가).
- 자원 용도:

| 자원 | 용도 |
|---|---|
| 고기·곡식 | 인구유닛 생산 (고기 5 + 곡식 5 → 유닛 1기) |
| 목재 | 연구 비용, (확장: 건물) |
| 철 | 군사 기술 연구 비용 |

## 6. 기술

기술 레벨 4계통, 각 5단계. 연구는 회의 턴에 예약 → 실행 턴에 자원 차감·완료.

| 계통 | 효과 | 비용(레벨 n) |
|---|---|---|
| 군사 | 전투력 +1/레벨 | 철 10×n |
| 채집 | 채취량 +1/레벨 | 목재 10×n |
| 이동 | 3레벨마다 이동력 +1 | 곡식 10×n |
| 인구 | 유닛 생산 비용 -10%/레벨 | 고기 10×n |

**기술력** = 군사 레벨 (전투 판정에 사용). **문명 점수** = 전 계통 레벨 합 + 인구 (세력 비율 판정에 사용).

## 7. 전투

같은 헥스에서 적대 유닛 조우 시:

- **기술력(군사 레벨) 동일** → 양측 유닛 모두 2턴 행동불능(채취·이동 불가). 유닛은 소멸하지 않음.
- **기술력 차이 ≥ 1** → 낮은 쪽 유닛 소멸, 높은 쪽 유닛 1턴 행동불능.
- 다대일: 유닛 수가 많은 쪽에 +1 보정 (수 우위 2배 이상일 때).
- 동맹끼리는 전투 불발, 같은 헥스 공존 가능.
- **다세력 전투(구현 확정)**: 헥스 단위로 세력별 전투력(군사 레벨 + 수 우위 보정)을 비교, 최고 전투력 단독 세력만 생존(패자 전멸, 승자 1턴 행동불능). 최고가 동률이면 해당 세력 전원 2턴 행동불능.
- **스턴 규칙(구현 확정)**: 행동불능 유닛만 있는 헥스에서는 전투가 재발동하지 않음(스턴 무한 갱신 방지). 행동불능 중에는 이동·채취 불가.

## 8. 인구·게임오버·점령

- 인구유닛 생산: 고기 5 + 곡식 5 소모, 자기 유닛이 있는 헥스에 생성. 상한 12기.
- **인구 0 = 게임오버(점령됨)**:
  - 마지막 유닛을 제거한 문명이 점령자가 됨.
  - 점령당한 플레이어는 관전이 아니라 **점령자 소속 유닛 중 점령자가 위임한 일부 유닛(기본 1기)** 을 조작하는 예속 상태로 계속 플레이.
  - 예속 문명의 채취분은 점령자에게 귀속.

## 9. 동맹과 흡수

- 회의 턴에 동맹 제안 → 상대 수락 시 성립. 탈퇴는 선언 후 다음 턴 발효.
- 동맹 상태에서는 상호 전투 불가, 시야 공유.
- **흡수 규칙**: 매 실행 턴 종료 시 동맹 쌍의 문명 점수 비율을 계산, **8:2를 초과**(강자 점수 ≥ 전체의 80%)하는 상태가 **3턴 연속** 지속되면 약자가 강자에게 흡수됨. 흡수된 플레이어는 점령과 동일한 예속 상태가 되며, 유닛은 강자 소속으로 편입.
  - 3턴 유예를 두는 이유: 순간 점수 변동으로 즉시 흡수되는 것 방지 + 약자에게 탈퇴 기회 제공.

## 10. 승리 조건 (제안)

다음 중 하나 달성 시 게임 종료:

1. **제패**: 예속 아닌 문명이 자신 1개만 남음.
2. **점수**: 제한 턴(기본 120턴 ≈ 80분) 도달 시 문명 점수 최고 문명 승리. 동맹은 공동 승리.

## 11. 네트워크 프로토콜 (WebSocket, JSON)

### 클라이언트 → 서버
| type | payload | 설명 |
|---|---|---|
| `join` | `{name}` | 입장, 국가 배정 요청 |
| `order.move` | `{unitId, path[]}` | 이동 명령 (회의 턴만) |
| `order.research` | `{branch}` | 연구 예약 |
| `order.spawn` | `{hex}` | 유닛 생산 예약 |
| `ally.propose / accept / leave` | `{civId}` | 외교 |
| `chat` | `{to, text}` | 전체/동맹/귓속말 |

### 서버 → 클라이언트
| type | 설명 |
|---|---|
| `welcome` | 배정 국가, 맵 마스크, 전체 초기 상태 |
| `phase` | 회의/실행 턴 전환 + 남은 시간 |
| `state` | 실행 턴 결과 diff (이동·전투·자원·기술·외교 변동) |
| `event` | 전투 발생, 점령, 흡수 등 알림 |
| `gameover` | 개인 게임오버 / 게임 종료 |

- 재접속: `join` 시 같은 세션 토큰이면 기존 문명 복귀. 30명 초과 시 관전자.
- 접속 끊긴 문명은 유닛이 제자리 채취 유지 (AI 대행 없음).

## 12. 배포

1. GitHub 저장소에 소스 push.
2. GitHub Actions: `docker build` → GHCR(GitHub Container Registry)에 이미지 push.
3. 리눅스 서버에서 `docker pull` + 재시작 (Watchtower 또는 Actions SSH 스텝).

```dockerfile
FROM node:22-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY . .
EXPOSE 3000
CMD ["node", "server/index.js"]
```

## 13. 개발 로드맵

| 단계 | 내용 |
|---|---|
| M1 | 맵 생성(worldmask) + Canvas 렌더링 + 접속/국가 배정 |
| M2 | 턴 루프 + 유닛 이동 + 자원 채취 |
| M3 | 전투 + 인구 생산 + 게임오버/점령 |
| M4 | 연구 + 동맹/흡수 + 승리 조건 |
| M5 | 채팅·UI 다듬기, 재접속, 배포 파이프라인 |

## 14. 미결정 사항 (확인 필요)

1. 승리 조건(§10)과 제한 턴 120턴이 적절한지.
2. 전투에서 기술 차이 시 "유닛 소멸" 규칙 동의 여부 (대안: 유닛이 후퇴).
3. 인구 상한 12기, 유닛 생산 비용(고기5+곡식5) 밸런스.
4. 흡수 판정의 "3턴 연속 유예" 동의 여부.
5. 예속 플레이어에게 위임되는 유닛 수(기본 1기).
6. 맵 크기 90×45 및 자원 분포 밀도.
